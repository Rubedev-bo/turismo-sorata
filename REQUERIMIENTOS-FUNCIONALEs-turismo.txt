REQUERIMIENTOS FUNCIONALES:
MÓDULO 1: CATÁLOGO DIGITAL DE EXPERIENCIAS
Visualización de Comunidades
* El sistema debe mostrar una página principal con tarjetas visuales de cada comunidad participante
* Cada tarjeta debe incluir: nombre de comunidad, imagen representativa y acceso a sus experiencias
Gestión de Fichas de Experiencias
* El sistema debe mostrar fichas individuales por cada experiencia turística
* Cada ficha debe contener: nombre, fotografía principal, descripción corta (máx. 200 palabras), duración estimada y precio en bolivianos
Sistema de Filtrado
* El sistema debe permitir filtrar experiencias por tipo de actividad (trekking, hospedaje, cultural)
* El sistema debe permitir filtrar experiencias por comunidad específica
* Los filtros deben ser acumulativos y mostrar resultados en tiempo real
Navegación entre Experiencias
* El sistema debe permitir navegar de la vista de catálogo a la ficha detallada de cada experiencia
* El sistema debe incluir un botón de retorno al catálogo desde cada ficha
MÓDULO 2: SISTEMA DE RESERVAS
Formulario de Reserva
* El sistema debe presentar un formulario con los siguientes campos obligatorios:
o Fecha de la experiencia (selector de calendario)
o Comunidad seleccionada
o Tipo de experiencia
o Número de personas (adultos/niños)
o Nombre completo del solicitante
o Correo electrónico
o Teléfono de contacto
Validación de Disponibilidad
* El sistema debe validar que la fecha seleccionada no sea anterior a la fecha actual
* El sistema debe mostrar mensaje de error si faltan campos obligatorios
Confirmación de Reserva
* El sistema debe generar un número único de reserva
* El sistema debe enviar correo electrónico de confirmación al turista con:
o Número de reserva
o Resumen de la experiencia seleccionada
o Datos de la reserva (fecha, personas, precio total)
o Información de contacto de la comunidad
Panel Administrativo de Reservas
* El sistema debe proporcionar un panel para administradores comunitarios
* El panel debe mostrar lista de reservas con: número, fecha, experiencia, estado, número de personas
* El sistema debe permitir cambiar estado de reserva (pendiente, confirmada, cancelada, completada)
* El sistema debe permitir filtrar reservas por fecha y estado
MÓDULO 3: GUÍA PRÁCTICA
Página "Cómo Llegar"
* El sistema debe mostrar una página con instrucciones de acceso en dos tramos:
o Tramo 1: La Paz ? Sorata (con opciones de transporte, horarios aproximados, costos)
o Tramo 2: Sorata ? cada comunidad (transporte local, tiempos, puntos de partida)
* Cada tramo debe incluir imágenes de referencia de terminales/puntos clave
Lista de Equipamiento
* El sistema debe mostrar listados de equipamiento recomendado por tipo de actividad
* Cada lista debe clasificar items en: esencial, recomendado, opcional
Información Climática
* El sistema debe mostrar información sobre:
* Clima promedio por temporada (lluviosa/seca)
* Mejor época para visitar cada tipo de experiencia
* Rango de temperaturas esperadas
* Recomendaciones de vestimenta

Caso de Uso No.4: Realizar Reserva
Actor: TuristaRequisitos de Usabilidad: 
* Formulario simple y claro con campos bien etiquetados
* Validación en tiempo real de campos obligatorios
* Calendario visual para selección de fecha
* Mensajes de error específicos y claros
* Confirmación visual antes de enviar
* Mensaje de éxito prominente con número de reservaPrecondiciones: 
* El turista debe haber consultado una experiencia específica
* El sistema de correo electrónico debe estar configurado y operativo
* La experiencia debe estar disponible para reservasDescripción:  
* El turista hace clic en el botón "Reservar ahora" desde la ficha de experiencia
* El sistema muestra formulario de reserva pre-rellenado con: comunidad y tipo de experiencia
* El turista completa los campos obligatorios:   
o Fecha de la experiencia (selector de calendario)  
o Número de personas (campo numérico con selectores de adultos/niños)  
o Nombre completo 
o Correo electrónico  
o Teléfono de contacto
* El sistema valida en tiempo real:  
o Que la fecha no sea anterior al día actual 
o Formato correcto de correo electrónico  
o Que todos los campos obligatorios estén completos
* El turista revisa los datos y hace clic en "Confirmar reserva"
* El sistema genera un número único de reserva (formato: RES-YYYYMMDD-XXXX)
* El sistema registra la reserva en base de datos con estado "Pendiente"
* El sistema envía correo electrónico de confirmación al turista conteniendo:   
o Número de reserva   
o Nombre de la experiencia   
o Comunidad  
o Fecha y número de personas   
o Precio total calculado  
o Información de contacto de la comunidad
* El sistema muestra pantalla de éxito con:   
o Mensaje de confirmación   
o Número de reserva destacado   
o Instrucción de revisar correo electrónicoExcepciones: 
Ex01: Si la fecha seleccionada es anterior a hoy, mostrar error: "La fecha debe ser igual o posterior a hoy" y resaltar campo en rojo
Ex02: Si faltan campos obligatorios al intentar confirmar, mostrar mensaje: "Por favor complete todos los campos obligatorios" y resaltar campos faltantes
Ex03: Si el formato de correo es inválido, mostrar: "Ingrese un correo electrónico válido"
Ex04: Si falla el envío de correo electrónico:   
* Registrar la reserva de todas formas   
* Mostrar advertencia: "Reserva registrada correctamente. No se pudo enviar el correo de confirmación. Número de reserva: [NÚMERO]. Por favor tome captura de pantalla"  
* Registrar error en log del sistema
Ex05: Si hay error al registrar en base de datos, mostrar: "Error al procesar la reserva. Intente nuevamente o contacte soporte"Postcondiciones: 
* El turista recibe confirmación por correo electrónico
* El turista tiene un número de reserva único para seguimiento
* La reserva es visible en el panel administrativo para el administrador comunitario

* Cambio de estado debe requerir confirmación

* El administrador debe tener permisos de gestión de reservas
* Debe existir al menos una reserva en el sistema
Descripción:

o Número de reserva  
o Fecha de la experiencia  
o Nombre del turista  
o Experiencia solicitada  
o Número de personas   
o Estado actual (Pendiente/Confirmada/Cancelada/Completada)   
o Acciones disponibles

o Por rango de fechas (desde - hasta)

Ex05: Si la sesión expiró, redirigir a login con mensaje: "Sesión expirada. Inicie sesión nuevamente"Postcondiciones: 
 El estado de la reserva se actualiza correctamente en el sistema
 Queda registro de auditoría del cambio (quién, cuándo, qué cambió)
